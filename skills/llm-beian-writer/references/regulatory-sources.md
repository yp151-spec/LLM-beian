# Regulatory Sources

Use these as starting points for source-grounded drafting. Re-check official pages before final submission because备案口径、地方要求和平台入口可能 change.

  - Government mirror: https://www.gov.cn/zhengce/202311/content_6917778.htm
- 国家网信办答记者问：《生成式人工智能服务管理暂行办法》适用范围、训练数据、服务规范、投诉举报等说明。
  - https://www.cac.gov.cn/2023-07/13/c_1690898326863363.htm

## Related Legal/Regulatory Families To Check

- 《网络安全法》《数据安全法》《个人信息保护法》：数据处理、个人信息、日志和安全管理基础。
- 《互联网信息服务深度合成管理规定》：深度合成服务和生成内容标识相关事项。
- 《互联网信息服务算法推荐管理规定》：算法备案、算法机制和用户权益相关事项。
- 《网络信息内容生态治理规定》：违法和不良信息治理口径。

## Source Hierarchy

1. Current official regulator pages and filing platform notices.
2. Laws, administrative regulations, departmental rules, national/industry standards and TC260 technical documents.
3. User's internal policies, contracts, vendor model documentation, test records, logs, screenshots.
4. Third-party articles and public templates, used only as drafting references, not as compliance authority.

## Citation Practice

- For submission drafts, cite document names and concrete evidence, not generic “according to regulations”.
- If using a standard or technical document, state `参考` unless it is mandatory for the specific filing path.
- If local regulator guidance differs from general sources, flag it as `属地口径待确认`.
