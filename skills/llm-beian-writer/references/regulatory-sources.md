## Related Legal/Regulatory Families To Check

- 《网络安全法》《数据安全法》《个人信息保护法》：data processing, personal information, logs, and security management.
- 《互联网信息服务深度合成管理规定》：deep synthesis services, result labels, technical management, and algorithm filing linkage for certain services.
- 《互联网信息服务算法推荐管理规定》：algorithm filing, algorithm mechanism review, user rights, algorithm safety responsibility.
- 《人工智能生成合成内容标识办法》 and generated/synthetic content labeling standards: explicit labels, implicit labels, metadata, user agreement notice.
- 《网络信息内容生态治理规定》：illegal and harmful information governance.
- Sector/local rules: education, medical, finance, water/utilities, state-owned enterprise, or other industry主管部门口径.

## Source Hierarchy

1. Current official regulator pages, filing platform notices, local regulator written feedback, and the user's received forms.
2. Laws, administrative regulations, departmental rules, national/industry standards, and TC260 technical documents.
3. User's internal policies, contracts, vendor model documentation, test records, logs, screenshots.
4. Prior client packages and third-party articles, used only as drafting references, not as compliance authority.

## Citation Practice

- For submission drafts, cite document names and concrete evidence, not generic “according to regulations”.
- If using a standard or technical document, state `参考` unless it is mandatory for the specific filing path.
- If local regulator guidance differs from general sources, flag it as `属地口径待确认`.
- Do not cite article numbers from memory. Verify the article text or quote from an official/user-provided source first.
